# Visual System Draft

## Presentation Identity

Title:

> Programming In The Language Of Intent

Mood:

- technical
- warm
- precise
- editorial
- quietly enthusiastic

Avoid:

- generic AI futurism
- neon brain imagery
- humanoid robots
- dark cyberpunk dashboards
- glossy corporate consulting cards

## Visual Motifs

- human-first source as a written artifact
- tangle as translation, compilation, or transformation
- intent flowing into implementation
- contracts becoming tests
- agents guided by explicit source material
- memory as a project notebook or indexed knowledge layer

## Palette Direction

Candidate palette:

- ink: near-black graphite
- paper: warm off-white
- code: muted blue-gray
- signal: soft green
- caution: warm amber
- accent: restrained electric cyan

Use color semantically:

- paper and ink for human-readable source
- blue-gray for implementation artifacts
- green for passing tests and verified contracts
- amber for questions, risk, or advisor moments
- cyan for agentic translation

## Typography Direction

Slide text should be sparse and readable.

Use large, open typography for claims. Use monospace only for source snippets, contracts, and code-like fragments.

Generated images should not rely on small text being legible. Any necessary precise slide text should be composed separately in the final slide image prompt with large, minimal wording.

## Composition Direction

Prefer:

- one dominant visual idea per slide
- editorial diagrams
- clean source-to-code transformations
- visual contrast between human-readable source and implementation artifact
- occasional practical workflow views for Antigravity

Avoid:

- repeated three-card layouts
- dense UI dashboards
- text-heavy generated images
- decorative orbs or abstract AI clouds
- screenshots that imply a real private project

## Image Prompt Pattern

Each image prompt should specify:

1. slide purpose
2. visual metaphor
3. objects and layout
4. palette
5. text to include, if any
6. text to avoid
7. aspect ratio: 16:9

Use the generated image as the actual slide, so visual prompts must leave safe space for readable title text when needed.
